Why a Personal Knowledge Vault?

The problem: AI conversations vanish. Your knowledge shouldn't.
THE PROBLEM
  • Every Claude session starts from zero — no memory of past decisions
  • Scattered notes across apps, repos, and chat histories
  • Knowledge gained in one project stays siloed from others
  • No institutional memory — same mistakes, same rediscoveries
THE SOLUTION
  • A structured Obsidian vault that Claude reads and writes to every session
  • Persistent memory that survives across projects and conversations
  • An AI Chief of Staff that maintains your knowledge base
  • Automated daily reflection that promotes insights and prunes noise

Setting Up Obsidian

The foundation: your markdown-powered second brain
INSTALL & CONFIGURE
  • Download Obsidian from obsidian.md — free for personal use
  • Create a new vault at ~/Documents/Vault/
  • Enable Settings → Files → Detect all file extensions
  • Enable Settings → Files → Default location for new notes → root
  • Turn on Settings → Core Plugins → Templates, Tags, Backlinks, Outgoing Links
OPTIONAL SETTINGS
  • Default view: Source mode (for YAML frontmatter editing)
  • Appearance: Dark theme + custom CSS (matches Claude's Navy theme)
  • Hotkeys: Cmd+N (new note), Cmd+O (quick open), Cmd+Shift+F (vault search)

Essential Obsidian Plugins

Community plugins that supercharge the Vault workflow
MUST-HAVE PLUGINS
  • Terminal — Run Claude Code directly inside Obsidian (Cmd+`)
  • Obsidian Web Clipper — Browser extension to capture web pages directly into 00_raw/
  • Excalidraw — Visual diagrams and sketches embedded in notes
  • Graph View — Explore connections between notes visually
NICE-TO-HAVE
  • Dataview — Query your vault like a database using YAML frontmatter
  • Templater — Auto-generate daily notes with correct frontmatter
  • Obsidian Git — Auto-commit and sync vault to GitHub
  • Calendar — Visual calendar for daily notes navigation
  • Linter — Auto-fix YAML frontmatter and formatting

Obsidian CLI

Control Obsidian from the terminal — perfect for Claude Code integration
INSTALLATION
  • Upgrade to the latest Obsidian installer (1.12.7+)
  • Go to Settings → General
  • Enable Command line interface
  • Follow the prompt to register the CLI in your shell
KEY COMMANDS
CommandDescription
obsidian open "Note Name"Open a note in Obsidian
obsidian create "Note" --content "..."Create a note programmatically
obsidian search "query"Full-text search across vault
obsidian list --folder 10_wikiList notes in a folder
CLAUDE INTEGRATION
  • Claude Code uses the CLI to create and manage notes when Obsidian is running
  • Configured in CLAUDE.md → Obsidian Integration section

Vault Folder Structure

Numbered prefixes keep folders in logical order
📁 Vault/
  • 📂 00_raw/ Drop zone — unprocessed files
  • 📂 10_wiki/ Processed knowledge base
    • 📂 11_entities/ People, companies, tools
    • 📂 12_concepts/ Ideas, frameworks, techniques
    • 📂 13_logs/ Activity log, meeting notes
      • 📄 log.md Chronological vault activity
    • 📂 14_daily/ Daily session notes
      • 📂 141_summaries/ Recaps from /reflect
    • 📄 index.md Master index
  • 📂 20_para/ Projects, Areas, Resources, Archive
  • 📂 30_zettelkasten/ Atomic, linked notes
    • 📂 31_permanent/ Promoted insights
  • 📂 90_meta/ Configuration and memory
    • 📄 user.md Owner profile
    • 📄 memory.md Decisions, feedback, context
DESIGN PRINCIPLES
  • Numbered prefixes (00_, 10_, 20_...) ensure consistent sort order
  • Each folder has a clear responsibility — no ambiguous dumping grounds
  • 00_raw → 10_wiki is the ingestion pipeline (Claude automates this)
  • 90_meta is the 'brain' — Claude reads it at every session start

Memory Architecture

Two-tier memory: chronological logs + synthesized permanent memory
Memory TypeDestinationMethod
User profile90_meta/user.mdUpdate sections in-place
Feedback90_meta/memory.mdSynthesize under ## Feedback
References90_meta/memory.mdAdd under ## References
Project logs13_logs/log.mdAppend dated one-liner
Insights31_permanent/*.mdAtomic note with frontmatter
KEY CONCEPT: COMPACTION
  • Daily notes accumulate verbose session context throughout the day
  • At session end, /reflect extracts the top 3 insights by lasting impact
  • Insights are promoted: to memory.md, user.md, or 31_permanent/ notes
  • Daily note is marked 'compacted' — original content preserved, summary added
  • Permanent memory stays concise while nothing is lost

Prompts — Initial Vault Setup

Copy-paste these prompts to bootstrap your vault
PROMPT 1: INITIALIZE THE VAULT
Create a Personal Knowledge Vault at ~/Documents/Vault/ with this structure: 00_raw/, 10_wiki/ (with 11_entities/, 12_concepts/, 13_logs/, 14_daily/, 14_daily/141_summaries/, index.md), 20_para/, 30_zettelkasten/31_permanent/, and 90_meta/ (with user.md and memory.md). Add YAML frontmatter to all .md files. Create a CLAUDE.md at the vault root that defines you as an AI Chief of Staff responsible for ingestion, bookkeeping, memory management, and Obsidian integration.
PROMPT 2: CONSOLIDATE EXISTING MEMORIES
Read all existing auto-memories from ~/.claude/projects/ across all project directories. Consolidate them into the Vault: user preferences → 90_meta/user.md, feedback/lessons → 90_meta/memory.md under ## Feedback, references → 90_meta/memory.md under ## References, project context → 90_meta/memory.md under ## Active Context. Synthesize duplicates — don't just concatenate.

Prompts — Skills, Hooks & Routing

Configure Claude to automatically maintain the vault
PROMPT 3: MEMORY ROUTING RULES
Add a "Memory Routing" section to ~/.claude/CLAUDE.md that overrides default auto-memory behavior. All memories must route to the Vault: user → 90_meta/user.md, feedback → 90_meta/memory.md ## Feedback, reference → 90_meta/memory.md ## References, project → 13_logs/log.md. Never create individual .md memory files or MEMORY.md index.
PROMPT 4: CREATE THE /REFLECT SKILL
Create a global skill at ~/.claude/skills/reflect/SKILL.md that performs end-of-day memory compaction: (1) gather today's modified files, (2) check for contradictions with permanent notes, (3) promote top 3 insights, (4) fill wiki gaps for unrepresented entities/concepts, (5) clean orphan notes, (6) generate a recap summary, (7) compact the daily note, (8) update the log.
PROMPT 5: ADD STOP HOOK REMINDER
Add a Stop hook to ~/.claude/settings.json that reminds me to run /reflect at the end of every session. Use an echo command with double quotes to avoid shell quoting issues.
