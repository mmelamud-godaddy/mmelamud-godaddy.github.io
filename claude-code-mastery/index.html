LEVEL 1: THE PROMPTER

"Claude is a tool"
YOU'RE HERE WHEN ...
  • You open Claude Code and immediately start prompting — no setup, no context
  • "Build me a landing page" — one shot
  • You see Claude as command > output
SKILLS TO MASTER
  • Writing clear, specific prompts
  • Reading and evaluating Claude's output
  • Basic terminal literacy
TRAP: Regression to the Mean
No context = Claude guesses everything.
Guessing = average of training data = slop.
This is where 90% of people live.

UNLOCK > Level 2:
Stop commanding. Start asking Claude to plan WITH you.
2

LEVEL 2: THE PLANNER

"Claude is a partner"
YOU'RE HERE WHEN ...
  • You use Plan Mode before building
  • You ask "What am I missing here?"
  • You discuss architecture before writing a single line of code
SKILLS TO MASTER
  • Plan Mode + extended thinking
  • Collaborative questioning: "What would an expert think here that I'm not?"
  • Letting Claude push back on your ideas
TRAP: Planning Blind
Great conversation, but Claude is still guessing about your project, your stack, your preferences. Words aren't enough.
Click to learn more
UNLOCK > Level 3:
The conversation doesn't matter if Claude doesn't have the right information.
3

LEVEL 3: THE CONTEXT ENGINEER

"You're not a coder — you're a curator"
YOU'RE HERE WHEN ...
  • You write and maintain CLAUDE.md files
  • You think about what Claude sees vs what it doesn't
  • You use /clear between tasks
SKILLS TO MASTER
  • Lean CLAUDE.md (not bloated)
  • Few-shot examples over long instructions
  • Context window management + /clear
  • Right context at the right moment
TRAP: Bloated CLAUDE.md
ETH Zurich: LLM-generated context files made agents WORSE in 5/8 settings. Less is more. (1 of 3)
Click to learn more
UNLOCK > Level 4:
You need capabilities beyond what Claude has natively. Time to go outside.
4

LEVEL 4: THE TOOLSMITH

"You equip Claude"
YOU'RE HERE WHEN ...
  • You leverage CLI tools Claude already knows (gh, playwright, contentful)
  • You install MCPs only when no CLI exists (Figma, Notion)
  • You think about token cost of every tool you add
SKILLS TO MASTER
  • CLI-first thinking (35x fewer tokens than MCP)
  • Knowing when MCP is the right choice
  • Understanding building blocks (frontend/backend/auth/data)
  • Asking Claude "explain HOW you did that"
TRAP: The Candy Shop
MCP schemas alone can consume 145K tokens. CLI does the same job in 4K. More tools ≠ better decisions. Less is more. (2 of 3)
Click to learn more
UNLOCK > Level 5:
You keep repeating the same workflows. What if Claude just... knew?
5

LEVEL 5: THE SKILLED

"Claude becomes YOURS"
YOU'RE HERE WHEN ...
  • You create custom skills that encode YOUR standards and workflows
  • Your Claude setup is unique to you
  • Nobody else's Claude works like yours
SKILLS TO MASTER
  • Skill authoring (frontmatter + supporting files)
  • Invocation control (who triggers what, when)
  • Subagent skills + dynamic context injection
  • Bundled skills (/batch, /simplify, /loop)
TRAP: Skill Overload
Too many skills = Claude can't pick the right one. 20–30 curated beats 1,000 generic. Less is more. (3 of 3)
Click to learn more
UNLOCK > Level 6:
One agent isn't enough. You need parallel execution.
6

LEVEL 6: THE ORCHESTRATOR

"You're not a curator — you're a manager"
YOU'RE HERE WHEN ...
  • You run multiple Claude sessions at once
  • You use worktrees for isolated agents
  • You decide WHAT gets done, not HOW
SKILLS TO MASTER
  • Task division + planning parallel work
  • Worktrees (--worktree / -w flag)
  • Multi-terminal session management
  • Agent Teams (the frontier)
TRAP: Skipping the Foundation
Without Levels 1–5, you get five agents producing five flavors of AI slop. Every level feeds this one.
